# HWTestAgent 프로젝트 요약 (경영진 보고용)

**작성일:** 2026-01-01
**프로젝트명:** HWTestAgent (Hybrid WorkHub Test Agent)
**담당:** Peter Chung

---

## 📌 프로젝트 개요

**목적:** 여러 WorkHub 시스템을 24시간 자동으로 테스트하고 문제를 사전에 발견·수정하는 AI 기반 테스트 자동화 시스템

**핵심 가치:**
- 사람이 없어도 PC가 꺼져도 테스트 실행
- 문제 발생 시 자동으로 수정 시도
- 새로운 문제 패턴을 스스로 학습하여 테스트 추가

---

## 🎯 주요 기능

### 1. 24/7 자동 테스트 실행
- **매일 2회** (오전 6시, 오후 6시) 자동 실행
- PC 종료 상태에서도 클라우드에서 테스트 계속 실행
- 테스트 대상: WBHubManager, WBFinHub, WBSalesHub, WBOnboardingHub
- GitHub Actions + Railway 하이브리드 아키텍처로 안정적 운영

### 2. 자동 문제 수정 (기본 활성화)
- 테스트 실패 시 원인을 자동으로 분석
- 다음 문제들을 **자동으로 수정**:
  - 환경 변수 누락 → Railway 환경 변수 자동 설정
  - 서비스 다운 → 자동 재시작 및 Health Check 대기
  - 데이터베이스 연결 문제 → Connection Pool 초기화
  - API 요청 제한 (Rate Limit) → Rate Limit 캐시 클리어
  - 보안 키 불일치 → Doppler 동기화 및 자동 재배포
- 수정 성공 시 자동으로 재테스트 (최대 3회 재시도)
- **70% 이상의 문제를 사람 개입 없이 자동 해결**

### 3. AI 자가 학습 기능 ⭐
- **자동 에러 감지**: 동일한 에러가 3회 이상 반복되면 자동으로 새로운 테스트 시나리오 생성
  - 에러 패턴 자동 분석 (에러 메시지, HTTP 상태 코드, 발생 빈도)
  - 새로운 테스트 케이스 자동 생성 및 YAML 시나리오 추가
- **테스트 자동 개선**: 실행 시간이 느린 테스트는 타임아웃 자동 조정
- **활용도 기반 시나리오 관리**:
  - 90일간 문제를 발견하지 못한 테스트는 자동 삭제
  - 활용도 점수로 테스트 가치 자동 평가 (100점 만점)
  - 점수 산정 기준: 실행 빈도(20%), 장애 발견율(50%), 최근성(20%), 신뢰도(10%)
- 매주 일요일 자정에 자동 유지보수 실행

### 4. 실시간 알림 및 리포팅
- **Slack 연동**: 테스트 결과를 실시간으로 Slack 채널에 전송
- **실패 알림**: 문제 발생 시 즉시 알림 (원인, 해결 방법, 영향 범위 포함)
- **자동 수정 알림**: 수정 성공 시 수정 내역 및 재테스트 결과 포함
- **주간 리포트**: 매주 월요일 오전에 지난 주 요약 리포트
  - 테스트 성공률 트렌드
  - 자동 수정 통계
  - 신규 발견 에러 패턴
  - 시나리오 활용도 분석

### 5. 웹 대시보드
- 최근 테스트 결과 조회 (실시간)
- 실패한 테스트 상세 로그 확인
- 자동 수정 이력 및 재시도 통계
- 수동 테스트 실행 버튼 (즉시 실행)
- 에러 패턴 분석 차트
- 시나리오 활용도 순위

## 💡 핵심 가치

### 개발자 생산성 향상
- 테스트 실행: **완전 자동화** (하루 2회 + 배포 시)
- 문제 발견: **사용자보다 먼저 감지**
- 야간/주말: **24시간 모니터링**
- 반복 작업: **수동 개입 불필요**

### 서비스 안정성 개선
- 문제 수정: **70% 이상 자동 해결**
- 장애 사전 발견으로 **서비스 다운타임 최소화**
- 새로운 에러 패턴 **자동 학습 및 대응**
- 불필요한 테스트 제거로 **효율적 테스트 관리**
